# Руководство пользователя URSUS SIEM

## Вход в систему

1. Откройте браузер и перейдите по адресу `http://<SIEM_HOST>:8080`
2. Введите логин и пароль, полученные у администратора
3. Нажмите **Войти**

> Если аутентификация прошла успешно, вы попадёте на главный дашборд.

---

## Главный дашборд

Дашборд показывает сводные метрики системы за последние 24 часа:

| Виджет | Описание |
|--------|---------|
| **Всего событий** | Общее число принятых логов |
| **Ошибки** | События с уровнем ERROR / CRITICAL |
| **Алерты** | Число открытых оповещений |
| **Агенты** | Количество активных агентов |
| **График активности** | Временной ряд событий по часам |
| **Топ хостов** | Хосты с наибольшим числом событий |
| **Топ уровней** | Распределение уровней критичности |

## Канал событий

**Меню: Мониторинг → Канал событий**

Канал — основной инструмент анализа событий в режиме реального времени.

### Выбор временного диапазона

Кнопки быстрого выбора: **15 мин | 1 ч | 2 ч | 6 ч | 12 ч | 24 ч | 7 д**

Для произвольного периода нажмите **Свой** и укажите даты вручную.

### Фильтрация PDQL

В строке **PDQL** введите запрос и нажмите **Применить** (или `Enter`):

```
# Только ошибки
filter(level = "ERROR") | sort(time desc)

# Конкретный хост за последний час
filter(host = "server01") | sort(time desc)

# Неудачные SSH-входы
filter(text contains "Failed password") | sort(time desc)
```

Полный синтаксис — см. [Руководство PDQL](pdql.md).

### Кнопка ⚡ (PDQL-редактор)

Открывает полноэкранный редактор для сложных многострочных запросов с подсказками.

### Автообновление

- **▶ Старт** — включить обновление каждые 10 секунд
- **■ Стоп** — остановить автообновление

### Группировка событий

Выберите одно или несколько полей в **Группировка** (Ctrl+клик для множественного выбора). Канал переключится в режим сводной таблицы: `поле | count`.

### Управление столбцами

- **Филдсет** — набор видимых столбцов (выберите из списка или создайте через ⚙)
- **⊟ Скрыть пустые** — убирает столбцы, в которых нет данных в текущей выборке; повторное нажатие восстанавливает все столбцы

### Сортировка

Кликните по заголовку столбца для сортировки. Повторный клик меняет направление (↑ / ↓).

### Детали события

Кликните на строку события — откроется боковая панель с полными данными:
- Все поля события с возможностью **+ фильтр** (добавить в PDQL)
- Кнопка **+ В инцидент** для привязки к инциденту

### Бесконечная прокрутка

Прокручивайте таблицу вниз — новые события подгружаются автоматически. Статус-строка показывает: `Загружено X из Y событий`.

### Экспорт

Кнопка **↓ Экспорт** → выберите формат:
- **CSV** — для Excel и BI-инструментов
- **JSON** — для скриптов и API
- **XML** — для интеграции с другими системами

## Live-логи

**Меню: Мониторинг → Live**

Отображает новые события в режиме реального времени (WebSocket). Новые строки появляются автоматически сверху.

Фильтры применяются через PDQL так же, как в канале событий.

---

## Поиск

**Меню: Поиск**

Полнотекстовый поиск по всем событиям. Введите ключевые слова и нажмите **Найти**.

Дополнительные фильтры: уровень, хост, агент, временной диапазон.

---

## Алерты

**Меню: Безопасность → Алерты**

### Просмотр алертов

Список пороговых правил и их срабатываний. Статусы: **Активен** / **Выключен**.

### Создание правила алерта

1. Нажмите **+ Добавить правило**
2. Заполните:
   - **Название** — понятное описание
   - **Условие** — SQL-подобное условие (например: `level = 'CRITICAL'`)
   - **Порог** — количество совпадений
   - **Окно (мин)** — временное окно
   - **Критичность** — LOW / MEDIUM / HIGH / CRITICAL
   - **Уведомить Webhook / Telegram** — способы доставки
3. Нажмите **Сохранить**

## Корреляция

**Меню: Безопасность → Корреляция**

### Просмотр правил

Список правил корреляции с полями: название, критичность, статус (включено/выключено).

### Редактирование правила

Нажмите на правило → откроется редактор с вкладками:
- **Основное** — название, критичность
- **Условия** — JSON-блок условий
- **SIGMA** — YAML-описание в SIGMA-формате

### Алерты корреляции

**Меню: Безопасность → Алерты корреляции**

Список всех срабатываний правил корреляции. Возможности:
- Фильтрация по статусу (OPEN / CLOSED)
- Закрытие алерта с примечанием
- Просмотр связанных событий

## Инциденты

**Меню: Инциденты**

Управление инцидентами безопасности.

### Создание инцидента

1. Нажмите **+ Создать инцидент**
2. Введите название, описание, приоритет
3. Нажмите **Создать**

### Привязка событий

В канале событий: выберите событие → **+ В инцидент** → выберите существующий инцидент.

### Жизненный цикл инцидента

`Открыт` → `В работе` → `Решён` → `Закрыт`

---

## Активы

**Меню: Активы**

### Вкладка Активы (хосты)

Инвентарь всех хостов. Для каждого хоста хранится:
- IP-адрес, ОС, подразделение, владелец
- Критичность (low / medium / high / critical)
- Теги для группировки
- Статус

Кнопки: **+ Добавить**, редактирование, удаление.

### Вкладка Агенты

Список подключённых агентов с:
- Статусом online/offline (по времени последнего события)
- Метриками CPU / RAM / диск
- Адресом и портом сервера подключения

---

## Учётные записи

**Меню: Активы → Аккаунты**

Справочник корпоративных учётных записей. Поля: имя, домен, роль, уровень риска, сервисный/привилегированный аккаунт.

## История запросов

В канале событий нажмите **↺ История** — откроется список последних PDQL-запросов. Нажмите на запрос, чтобы применить его повторно.

## Филдсеты (наборы столбцов)

Филдсет определяет, какие столбцы отображаются в таблице событий.

### Создание филдсета

1. В канале событий нажмите ⚙ рядом с выбором филдсета
2. Нажмите **+ Создать**
3. Введите название
4. Отметьте нужные поля
5. Нажмите **Сохранить**

### Применение

Выберите филдсет из выпадающего списка — таблица событий обновится.

## Состояние системы

**Меню: Система → Состояние**

Метрики производительности сервера:
- CPU, RAM, диск
- Число активных агентов
- Скорость приёма событий (events/sec)
- Время работы (uptime)

## Часто задаваемые вопросы

**Q: Почему нет событий в канале?**
- Проверьте временной диапазон (возможно, выбран слишком короткий)
- Проверьте PDQL-фильтр — возможно, условие слишком строгое
- Убедитесь, что агенты подключены (Активы → Агенты)

**Q: Как найти все события с конкретного IP?**
```
filter(src.ip = "192.168.1.100") | sort(time desc)
```

**Q: Как посмотреть только критические события за сегодня?**
- Выберите период **24 ч**
- Введите PDQL: `filter(level = "CRITICAL") | sort(time desc)`

**Q: Алерт пришёл, но я не понимаю что произошло?**
1. Откройте алерт в **Корреляция → Алерты корреляции**
2. Посмотрите на имя правила и условия
3. В канале событий выполните запрос по паттерну из условия правила
4. Изучите события в указанный временной промежуток

**Q: Как экспортировать все события за неделю?**
1. Выберите период **7 д**
2. Нажмите **↓ Экспорт → CSV**
> Для больших объёмов данных используйте API с постраничной загрузкой.
