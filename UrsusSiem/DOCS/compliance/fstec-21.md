# Соответствие Приказу ФСТЭК России №21

## Что это

Приказ ФСТЭК России от 18.02.2013 №21 «Об утверждении Состава и
содержания организационных и технических мер по обеспечению безопасности
персональных данных при их обработке в информационных системах
персональных данных» — основной нормативный документ для коммерческих
ИСПДн в РФ.

Применяется ко всем компаниям, обрабатывающим **персональные данные**
(сотрудники, клиенты, посетители сайта). То есть — почти ко всем.

## Что закрывает URSUS

URSUS реализует **технические меры регистрации событий безопасности** —
группа `РСБ` и часть `ИАФ`/`УПД`. Полное соответствие зависит от
организационных мер, которые URSUS не делает (политики, обучение,
паспорта ИСПДн).

| Группа мер | URSUS закрывает | Чем именно |
|---|---|---|
| ИАФ.1 — Идентификация пользователей | ✅ | Логи аутентификации, поиск, отчёт |
| ИАФ.5 — Регистрация событий ИБ | ✅ | `audit_log` table, immutable |
| УПД.1 — Управление учётными записями | ✅ | RBAC + аудит изменений |
| УПД.4 — Разделение полномочий | ✅ | Роли admin/operator |
| РСБ.1 — Определение событий | ✅ | 60 SIGMA-правил из коробки |
| РСБ.3 — Сбор и хранение событий | ✅ | DuckDB / Postgres / ClickHouse |
| РСБ.5 — Защита информации о событиях | ✅ | Audit-log only INSERT, бэкап в S3 |
| АНЗ.1 — Выявление уязвимостей | ⚠️ Частично | События об эксплойтах, но не active scan |
| ОЦЛ.1 — Контроль целостности ПО | ⚠️ Частично | File integrity monitoring (Sprint 11 — v2.2) |
| ОДТ.5 — Резервное копирование | ⚠️ Документация | `cp data.duckdb` + cron |

**Не закрывает** (требуется отдельно):
- Организационные меры (политики, инструкции, обучение)
- ЗТС (защита технических средств — физическая безопасность)
- ЗИС (защита информационной системы — это про периметр, не SIEM)
- Антивирусные средства (AVZ)

## Как сгенерировать отчёт

### Через UI

System → Compliance → выбрать «Приказ ФСТЭК №21» → нажать «Сформировать
отчёт за квартал» → скачать PDF.

### Через API

```bash
# Предпросмотр (JSON — для проверки данных)
curl -s -H "Authorization: Bearer $JWT" \
  https://siem.example.com/api/compliance/fstec-21/preview | jq

# PDF
curl -s -H "Authorization: Bearer $JWT" \
  https://siem.example.com/api/compliance/fstec-21/pdf?from=2026-01-01&to=2026-03-31 \
  -o report-q1-2026.pdf
```

## Что подписывать

URSUS генерирует **технический отчёт со свидетельствами**. Это **не**
сертификат соответствия. Для прохождения официальной проверки:

1. Скачать PDF из URSUS.
2. Сопоставить с **Перечнем мер защиты** в Приказе №21.
3. Подписать у CISO/ИТ-директора как «отчёт о работе технических средств
   защиты».
4. Подкрепить организационными мерами (приказы, инструкции).

## Сертификация СЗИ

URSUS **не сертифицирован ФСТЭК как СЗИ** в v2.0. Это отдельный
процесс:
- Стоимость: 500K-1.5M ₽
- Срок: 6-12 месяцев
- Класс защиты: К4 (стартовый) → К1
- Делается одним юр.лицом, не сообществом

После сертификации URSUS можно будет использовать в ГИС и КИИ. До этого:
- Коммерческие ИСПДн — да, URSUS подходит
- ГИС / КИИ — нужно сертифицированное СЗИ + ваша оценка пригодности
